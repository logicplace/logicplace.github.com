Introduction

The use of this module can be as complex as you want it to be. Simple use is akin to all dice rollers I've seen, and based on standard notation for those who do this sort of thing written. That is, of course, 1d6 being "roll one six-sided die". It may look something like this:

dice 1d6 Set 1: [ 4 ] = 4

Where complex use can be something like:

dice --p 5,Will Save: + 1d10-z1 >> 0?Successful (by %) % s:Failed Set 1: [ 6 ] = 6 Will Save: Successful (by 1)

You may also use d instead of dice for the command.

Understanding Output

I tried to make the output as easily understandable as possible, but nonetheless here is a quick explanation.

Large integers are expressed in #e+# notation, entire output string is truncated to 100 characters.

First it outputs all of the dice rolls that were made with its set number. If the roll was in a conditional, it will not report that set if it wasn't rolled (this does not affect numbering).

Its display is in the form Set #: [ Rolls ] = Total

If used from an each statement, form is: Set #: Totals

The set number is just a number, starting at 1.

Rolls is a comma separated list of the rolls made and any calculations performed on them individually. If a roll value's representation is exactly the same as the previous, it will output a period instead. This is probably the most confusing part of the output, so remember it.

Total is simply all the rolls added together, in sequence.

Afterwards is the expression output. Multiple expressions are separated by commas on input. This output is one expression per line. By default, it output the calculation and its result separated by an equals sign. If the expression and the result look the same, it does not output anything for this expression, assuming the output is already obvious. There are command line options to change the output.
