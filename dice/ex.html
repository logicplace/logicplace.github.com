For loop:

dice --v 5d1e+s?s Set 1: [ 1 + 0 ? 1, 1 + 1 ? 2, 1 + 2 ? 3, 1 + 3 ? 4, 1 + 4 ? 5 ] = 15

This abuses conditionals to set s to an incrimenting number.

Ignore this set:

1d2+123 > 125 0

Essentially, with the way processing works, stating a number with no operators silently overwrites the current set value and expression output, making it not output on the grounds that 0 looks like 0.
